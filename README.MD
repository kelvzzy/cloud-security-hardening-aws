# AWS Cloud Security Hardening Project

## Overview
This project demonstrates how insecure cloud configurations can expose systems to threats, and how to properly secure them using AWS best practices.

## Objectives
- Identify misconfigurations
- Apply security hardening techniques
- Demonstrate before/after states

## Tools Used
- AWS EC2
- IAM
- CloudWatch

## Key Learnings
- Importance of least privilege
- Network access control
- Monitoring and logging
